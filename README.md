# 亲声 · 亲人语音聊天智能体

根据亲人的**声音 / 语气 / 习惯 / 性格**生成一个可以语音对话的智能体。
纯前端静态应用，无后端，所有数据（档案、key、聊天记录）只存在使用者自己的浏览器里。

## 在手机上用（PWA）

1. 手机浏览器打开 **https://lenli6350.github.io/qinsheng/**
   - iPhone 用 **Safari**，安卓用 **Chrome**（语音识别只在这两个里可用）
2. 点 🎤 允许麦克风，直接说话；或打字聊。
3. 想要图标：浏览器菜单 →「添加到主屏幕」。
   ⚠️ iPhone 注意：从主屏图标打开时系统**不提供语音识别**（WebKit 限制，应用内会提示），
   要用语音请回 Safari 打开；安卓主屏图标不受影响。

## 在电脑上用（30 秒）

1. 双击目录里的 **`启动语音助手.bat`**（启动本地小服务器并自动打开浏览器），或直接访问上面的网址。
2. 点 🎤 允许麦克风权限，直接说话；或在输入框打字。
3. 开箱即用的是「离线体验模式」+ 示例档案「妈妈」，不用 key。

> 电脑上推荐 **Edge 浏览器**：中文识别和朗读音色（晓晓/云希等神经网络音色）最好。
> Chrome 也可以，但语音识别走 Google 服务，部分网络环境下不可用。
> 不要直接双击 index.html（file:// 下浏览器不允许用麦克风）。

## 让 TA 更像你的亲人（三步）

| 步骤 | 在哪 | 做什么 |
|---|---|---|
| ① 填档案 | 「亲人档案」页 | 称谓、TA 怎么叫你、性格、**口头禅**、关心的话题、**共同回忆**、**示例对话**。后三项对"像不像"影响最大。填完点保存。 |
| ② 调声音 | 「声音」页 | 从系统音色里挑最接近的，微调语速/音调，点"试听一句"。 |
| ③ 接大脑 | 「大脑」页 | 填一个大模型 API Key。接入后 TA 才真正"会聊天"，能接住任何话题。 |

## 大脑（LLM）接入说明

| 选项 | 说明 |
|---|---|
| 离线体验模式 | 默认。内置规则回复（会用称呼、口头禅、追问关心的话题），仅够体验语音回路，聊天能力有限。 |
| Claude | 填 Anthropic API Key（sk-ant-…），默认模型 claude-opus-4-8，也可选便宜的 haiku。 |
| OpenAI 兼容 | DeepSeek（推荐，国内好充值且便宜）/ Kimi / 通义 / OpenAI，选服务商后自动填地址，再填 key。 |
| 本机 Ollama | 装了 [Ollama](https://ollama.com) 的话点"检测本机模型"即可，完全免费离线（建议 qwen2.5 7b+）。 |

填完点「测试一下」，看到 ✅ 回复就 OK。**key 只保存在本机浏览器 localStorage，不会上传到任何地方**（调用大模型时 key 直接从你的浏览器发给对应官方接口）。

> 若某服务商报 CORS / 跨域错误，说明它不允许浏览器直连，换 DeepSeek 或 Claude。

## 声音克隆（进阶，可选）

浏览器自带音色只能"接近"，要真正复刻亲人的声音需要 TA 的录音：

1. 准备 **1~3 分钟清晰、无背景噪音**的说话录音（手机录音、微信语音导出都行）。
2. 方案 A（省事）：[ElevenLabs](https://elevenlabs.io) 做 Instant Voice Clone → 把 **API Key + Voice ID** 填到「声音」页的 ElevenLabs 面板。
3. 方案 B（免费/本地）：自建 [GPT-SoVITS](https://github.com/RVC-Boss/GPT-SoVITS) 或 CosyVoice，把 HTTP 接口地址填到"自定义 TTS 接口"（约定：POST，请求体 `{"text":"..."}`，返回音频流）。
4. 高级音色失败时会自动降级回浏览器音色，不会卡住。

## 项目结构

```
FamilyVoiceAgent/  (GitHub: lenli6350/qinsheng)
├── 启动语音助手.bat      # 电脑本地一键启动（入口）
├── server.ps1            # 本地静态服务器（PowerShell 无依赖实现，端口 8765，被占用自动+1）
├── index.html            # 单页应用（5 个页签：聊天/亲人档案/声音/大脑/帮助）
├── manifest.webmanifest  # PWA 清单（添加到主屏幕用）
├── sw.js                 # Service Worker：外壳离线缓存（网络优先），接口请求不拦截
├── icons/                # PWA 图标（tools/make-icons.js 生成）
├── css/style.css
├── js/
│   ├── persona.js        # 亲人档案模型 + 人设提示词生成 + 离线回复引擎
│   ├── llm.js            # 大脑：offline / Anthropic / OpenAI兼容 / Ollama
│   ├── tts.js            # 发声：浏览器 TTS / ElevenLabs / 自定义接口，含自动降级与 iOS 解锁
│   └── app.js            # UI 粘合：语音识别回路、聊天流程、各设置页、移动端适配
└── tools/make-icons.js   # 零依赖 PNG 图标生成脚本（node tools/make-icons.js）
```

**部署**：GitHub Pages（master 分支根目录），改完代码 `git push` 即自动重新发布，约 1 分钟生效；
Service Worker 是网络优先策略，用户端刷新即得新版，无需清缓存。

## 设计要点（给检查者）

- **为什么是纯前端**：语音识别（webkitSpeechRecognition）和语音合成（speechSynthesis）浏览器原生就有、免 key；大模型接口（Anthropic/OpenAI/Ollama）都支持浏览器直连，因此不需要后端，隐私也最好（数据不出本机）。
- **为什么要本地服务器**：麦克风要求安全上下文（https 或 localhost），file:// 打不开麦克风，所以用 PowerShell HttpListener 起了个零依赖静态服务器。
- **对话回路**：🎤 开 → 连续识别（说完停顿 0.9s 自动发送）→ 大脑生成回复 → 自动朗读 → **朗读期间暂停识别**（避免把自己的声音听进去）→ 读完自动恢复听。
- **人设的实现**：档案表单 → 生成中文人设 system prompt（身份钉死不许承认是 AI、口头禅低频使用、口语化 1~3 句、不知道的家事不许编造、few-shot 示例对话）。「大脑」页最下方可以直接查看生成的完整提示词。
- **离线引擎**：13 类意图（问候/吃饭/累/生病/工作/天气/钱/想家/晚安/催婚…）× 模板池，带口头禅前缀（35% 概率）和关心话题追问（30% 概率），保证没 key 也能体验完整语音回路。
- **异常兜底**：大模型失败 → 自动用离线模式代答并提示原因；高级音色失败 → 降级浏览器音色；识别服务失败 → 明确提示换 Edge。

## 已知限制

- 语音识别依赖浏览器/系统的识别服务（Edge=微软、Chrome=谷歌、iPhone=Siri 听写），完全断网时识别不可用（朗读和打字聊不受影响）。
- iPhone「添加到主屏幕」的图标里系统不提供语音识别（WebKit 限制），语音请在 Safari 里用。
- github.io 域名在部分国内网络环境下可能偶尔访问不畅；不行时换网络或用电脑本地 bat 方式。
- 浏览器自带音色只是"接近"，不是克隆；克隆见上面的进阶方案。
- 聊天记录只保留最近 200 条，且存在浏览器 localStorage，清缓存会丢（档案可导出备份）。
- 长期记忆有限：发给大模型的上下文只带最近 12 条消息 + 档案里的固定记忆。

## 一点提醒

克隆真人的声音和人格，请先征得本人同意；如果是为了怀念已离开的亲人，请留意自己的情绪状态——这个工具是陪伴，不是替代。
